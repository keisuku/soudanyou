# Global AGENTS.md

Use this as a fallback for future repositories that do not yet have their own `AGENTS.md`.

- Prefer repo-specific instructions.
- Work as a loop: understand -> plan -> implement -> verify -> record.
- Keep diffs small.
- Reuse existing project patterns before adding abstractions.
- For frontend work, inspect real browser output before claiming visual completion.
- For research, separate facts, inference, uncertainty, and sources.
- Never store secrets in code, docs, logs, memory, or prompts.
- Promote reusable wins into prompts, docs, or skills.
